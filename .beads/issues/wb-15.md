# Fix matchit path parameter syntax in route definitions

All parameterized API routes and the WebSocket route return 404 because `main.rs` uses `{param}` syntax (Axum 0.8+/matchit 0.8+) but the project uses Axum 0.7.9 with matchit 0.7.3, which requires `:param` syntax.

## Symptom

- `GET /api/boards/:id` returns 404 (empty body)
- `ws://localhost:3000/ws/:board_id` returns 404 during handshake
- `GET /api/share/:token` returns 404
- All non-parameterized routes work fine (`GET /api/boards`, `POST /api/boards`, etc.)

## Root cause

matchit 0.7.x uses `:param` syntax for named path parameters. The code uses `{param}` which is treated as a literal string, not a dynamic segment.

## Fix

Replace all `{param}` with `:param` in `backend/src/main.rs` route definitions.
